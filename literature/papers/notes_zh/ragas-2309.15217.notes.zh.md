# RAGAS：面向 RAG 的无参考自动评测框架

> 来源：literature/papers/markdown/ragas-2309.15217.md
> 说明：这是中文精读笔记，不是全文翻译。

## 一句话定位

RAGAS 提出一组不依赖标准答案的 RAG 自动评测指标，用 LLM 对“回答是否忠于上下文、是否回答了问题、检索上下文是否聚焦”做可解释打分。

## 研究问题

RAG 系统常见评测依赖人工标注、参考答案或模型内部概率，难以用于只有 API 访问、没有标准答案的实际系统。论文要解决的问题是：在给定问题、检索上下文和生成回答的情况下，如何自动判断 RAG 输出是否可靠，并能定位问题来自检索还是生成。

## 评测对象与任务设置

标准设置是三元组：问题 `q`、检索上下文 `c(q)`、系统回答 `a(q)`。评测对象不是单一模型，而是完整 RAG 流程中的两个核心环节：检索出的上下文是否有用且少噪声，生成回答是否基于上下文并真正回应问题。

## 指标与评测方法

核心指标有三类。Faithfulness 衡量回答中的陈述能否从上下文推出：先让 LLM 将回答拆成若干原子陈述，再逐条判断是否被上下文支持，最终用“被支持陈述数 / 总陈述数”作为分数。Answer relevance 衡量回答是否针对原问题：从回答反向生成多个可能问题，再比较这些问题与原问题的嵌入相似度，平均相似度越高代表回答越贴题。Context relevance 衡量上下文是否聚焦：让 LLM 从上下文中抽取回答问题所必需的句子，用“必要句子数 / 上下文总句数”惩罚冗余上下文。

方法上，RAGAS 强调 reference-free，不要求标准答案；同时用结构化提示把评测任务拆成抽取、验证、相似度计算等子步骤，而不是直接让 LLM 给总分。

## 数据集/标注/实验设计

论文构造 WikiEval：选取 50 个 2022 年以后、近期有编辑的 Wikipedia 页面，围绕页面导语生成问题和回答，并为三类质量维度构造对比样本。Faithfulness 通过比较“带上下文回答”和“无上下文回答”来得到人工偏好；Answer relevance 通过生成不完整回答作为负例；Context relevance 通过加入来自反向链接或补全文本的额外句子制造冗余上下文。两名英文流利标注者对样本做偏好判断，Faithfulness 和 Context relevance 一致率约 95%，Answer relevance 约 90%。

实验用 pairwise accuracy 衡量自动指标与人工偏好是否一致，并与两类基线比较：直接让 ChatGPT 打 0-10 分的 GPTScore，以及让 ChatGPT 在两个候选中排序的 GPT Ranking。

## 主要发现

RAGAS 与人工偏好一致性明显高于直接打分或直接排序。表中结果显示，Faithfulness 上 RAGAS 约 0.95，GPTScore 约 0.72，GPT Ranking 约 0.54；Answer relevance 上 RAGAS 约 0.78，高于 0.52 和 0.40；Context relevance 上 RAGAS 约 0.70，高于 0.63 和 0.52。论文也指出 Context relevance 最难，尤其长上下文中抽取“关键句”时 LLM 容易不稳定。

## 局限与风险

RAGAS 仍依赖 LLM 判断，可能继承 judge 的偏见、格式错误和上下文长度敏感性。Answer relevance 的反向问题生成加嵌入相似度无法直接判断事实正确性；Context relevance 用句子比例评估会受句子切分和上下文粒度影响。它评估的是单轮三元组，不直接覆盖检索召回、工具调用、多轮状态和真实用户成本。

## 对本仓库 ragbench 的启发

可以把 RAGAS 作为最小可运行的无参考评测基线：每条样本只需 `question/context/answer`，适合快速批量筛查。仓库中的 ragbench 可将三项指标拆成独立报告，分别归因于检索冗余、回答跑题和回答不忠实，而不是只给一个总分。

## 对“公理化 LLM wiki”论文的可借用点

可借用“先拆原子陈述、再逐条验证”的公理化思路：Wiki 条目中的每个事实断言都应可追溯到给定证据；回答相关性可改写为“生成内容是否满足用户查询意图”；上下文相关性可改写为“引用证据是否为证明该断言所必需”。这能把 wiki 生成从整体主观评分转为断言级可审计评测。

## 可落地的测试项

- 断言忠实度测试：把回答拆成原子声明，逐条检查是否被检索片段支持，输出 unsupported claim 列表。
- 回答跑题测试：对回答反向生成 3-5 个问题，与原问题做语义相似度阈值判断。
- 上下文冗余测试：统计被判定为必要的句子比例，并记录未使用片段占比。
- 长上下文压力测试：把关键证据放在开头、中间、结尾，比较 Faithfulness 和 Context relevance 是否下降。
- 负例回归测试：加入无关但主题相近的上下文，检查回答是否被误导。
- 三维归因报告：每个失败样本标注为检索噪声、检索缺失、生成幻觉或回答不完整。
